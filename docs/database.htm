Database Structure as of beta v3

usr - user's table

usr - ingame user name
password - user's password
usrname - user's real name
usremail - users email address
usraddress - users home address (may be used for prizes later on)
usrcountry - users country
usrcompany - users comapany name
usrstatus - the status of the user ie bankrupt (0), normal (1), moderater (2), admin (3), superadmin(4)
level - the users level, this will allow them to build bigger buildings and new buildings
money - the users money

buildinglocation - the table for the location of buildings

town - the town id where the building is from
vert - verticle location of the building in the town
horz - horizontal location of the building in the town
buid - the building id from the buildingtype table
usr - the owner of the building
prod - % of work being done in the building
hire - the hire rate of the building
p1p - work rate from product 1 (can't be higer than prod) 0 if there is no product
p2p - same as p1p but for product 2
p3p - for product 3
p4p - for product 4
p5p - for product 5

buildingtype - all the differrent types of building

buid - building id number
type - type of building eg commerce, industry, hq, res etc etc
name - name of the building
image - location of small image of building
bimage - location of big image of building
upgradeid - buid of the next upgrade eg for hq 1 it will be for hq 2
level - level that user must be to build this type
cost - cost of building
m1 - product 1 made (for indy)
m2 -
m3 -
m4 -
m5 -
m1m - amount of product 1 make per % of production
m2m -
m3m -
m4m -
m5m -

products -all the differrent type fo products avaible

pid - product id
pname - product name
cost - cost of product at 100% price
r1 - pid of product 1 required to make this product (0 if not needed)
r1a - amount required of r1 to make 1 of the products (0 if not needed)
r2 to r5a are the same only for each type of product needed

rank - the ranking system, this is still in the early stages and has a lot of work to be done on it so i'm not sure what the db structure will end up being

usr - user name
orank - overall rank

towns - all the towns in the world

tid - town id
tname - town name
owner - the mayor or president's user name (none for no mayor/pres)
ttype - town type either a normal town or capitol
tall - town alliance, which presidency the town is ruled under (0 for none) will be used later in war module
money - the towns money
taxr - res tax
taxc - commerce tax
taxi - industry tax
taxp - presidential tax (only pres can change)
lc - number of lc population
lce - employed lc
mc - middle class population
mce - employed mc
hc - high class population
hce - employed hc
fa - famous people population
fae - employed famous people
cost - land cost per square
